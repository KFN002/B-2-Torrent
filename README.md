# B-2-Torrent

Privacy-first, self-hosted BitTorrent management app for laptops and PCs with a Next.js frontend, Go backend, Docker deployment, Tor proxy support, strict local defaults, and helper utilities for security-focused workflows.

Use B-2-Torrent only for lawful content you have the right to download or share. Privacy tooling reduces exposure, but it cannot guarantee anonymity against every network, endpoint, provider, or device-level threat.

## Project Layout

```text
frontend/       Next.js app, UI components, frontend Dockerfile, frontend env
backend/        Go REST API, torrent client integration, backend Dockerfile
nginx/          Local reverse proxy for frontend and /api
browser/        Optional standalone secure browser Electron app
vpn-client/     Optional standalone VPN/proxy Electron app
scripts/        install, health, and environment generation scripts
```

All Next.js frontend files now live under `frontend/`. The root `package.json` is only an orchestration entrypoint.

## Quick Install

### macOS

```bash
./scripts/install-macos.sh
docker compose up -d
open http://localhost
```

### Windows PowerShell

```powershell
.\scripts\install-windows.ps1
docker compose up -d
start http://localhost
```

### Manual

```bash
node scripts/gen-env.mjs
corepack enable
pnpm --dir frontend install --frozen-lockfile
docker compose build
docker compose up -d
```

## Environment

Generate secrets and local defaults:

```bash
npm run gen-env
```

This creates:

- `.env` for Docker Compose secrets and ports.
- `frontend/.env.local` for public frontend settings.

Re-generate with:

```bash
node scripts/gen-env.mjs --force
```

## Running

```bash
docker compose up -d
npm run health
docker compose logs -f backend
docker compose down
```

Default URLs:

- App via Nginx: `http://localhost`
- Frontend direct: `http://localhost:3000`
- Backend health: `http://localhost:8080/api/health`
- RabbitMQ management: `http://localhost:15672`

## Security And Privacy Defaults

- Docker services bind host ports to `127.0.0.1` by default.
- Database and RabbitMQ passwords are generated by `scripts/gen-env.mjs`; insecure defaults were removed from Compose.
- Frontend telemetry was removed and Next telemetry is disabled in Docker.
- External IP/geolocation leak checks are off by default. Set `NEXT_PUBLIC_ENABLE_EXTERNAL_LEAK_CHECKS=true` only when you deliberately want the UI to call third-party leak-check services.
- Nginx access logs are disabled and backend logs default to warnings/errors.
- Backend CORS uses an allowlist from `CORS_ALLOWED_ORIGINS`.
- File encryption and secure deletion endpoints are restricted to app-owned download, upload, and temp directories.
- Tor proxy routing is enabled in Docker with `PROXY_CHAIN=tor:9050`; use a trusted VPN at the OS/network layer when your threat model requires it.

## Deployment Notes

Before deploying outside localhost:

1. Replace localhost-only port bindings in `docker-compose.yml` with your intended network exposure.
2. Put TLS in front of Nginx or terminate TLS at your platform load balancer.
3. Keep `.env` out of source control and rotate generated secrets for each environment.
4. Use persistent, encrypted storage for `DOWNLOAD_DIR` when downloads must survive restarts.
5. Review local law, tracker policies, and content rights before enabling torrent workloads.
